# HealthFit Frontend UI/UX 개선 변경점

이 문서는 HealthFit 프론트엔드에서 진행한 UI/UX 개선 1~10단계 작업 내용을 정리한 문서입니다.

## 1단계: 상단 네비게이션 정리

- 주요 메뉴를 `식단`, `운동`, `기록`, `커뮤니티`, `마이` 중심으로 재정리했습니다.
- AI 상담 메뉴는 별도 강조 링크로 분리했습니다.
- 로그인/로그아웃/회원가입 영역의 문구와 배치를 정리했습니다.
- 모바일 화면에서 상단 메뉴가 가로 스크롤로 자연스럽게 보이도록 보완했습니다.

## 2단계: 기록 메뉴 통합

- `RecordsView.vue`를 추가해 기록 허브 화면을 만들었습니다.
- 식단 기록, 운동 기록, 신체 기록, 진행 현황으로 바로 이동할 수 있는 구조를 만들었습니다.
- 상단 네비게이션의 `기록` 메뉴가 `/records`로 이동하도록 라우터를 추가했습니다.

## 3단계: 대시보드 첫 화면 강화

- 홈 화면에서 오늘의 식단, 운동, 체중 변화 요약을 보여주도록 개선했습니다.
- 로그인 사용자는 오늘 기록 기반 요약을 확인할 수 있게 했습니다.
- 식단 기록, 운동 기록, 진행 현황으로 바로 이동하는 빠른 행동 버튼을 추가했습니다.

## 4단계: AI 기능 비활성 상태 표시

- AI 상담, 식단 추천, 식단 평가, 운동 추천 화면에 `AI 준비 중` 상태를 명확히 표시했습니다.
- 아직 백엔드 AI 연동이 완료되지 않은 기능은 예시/목업 동작임을 화면에서 알 수 있게 했습니다.
- 추후 OpenAI API 연동 시 교체해야 할 영역을 분리해두었습니다.

## 5단계: 삭제 버튼 UX 개선

- 삭제 버튼을 즉시 삭제 방식에서 2단계 확인 방식으로 변경했습니다.
- 적용 화면:
  - 식단 기록 삭제
  - 운동 기록 삭제
  - 신체 기록 삭제
  - 커뮤니티 댓글 삭제
- 첫 클릭 시 `확인 삭제` 상태로 바뀌고, 한 번 더 눌러야 실제 삭제됩니다.
- `취소` 버튼을 추가해 실수로 삭제 버튼을 누른 경우 되돌릴 수 있게 했습니다.

## 6단계: 저장 성공 피드백 강화

- 공통 토스트 알림 구조를 추가했습니다.
  - `src/stores/toast.js`
  - `src/components/common/ToastHost.vue`
- App 전체에 토스트 표시 영역을 연결했습니다.
- 성공 알림을 적용한 흐름:
  - 회원가입 완료
  - 식단 기록 저장
  - 운동 기록 저장
  - 신체 기록 저장
  - 프로필 저장
  - 게시글 작성
  - 댓글 작성
- 폼 하단 메시지와 화면 우측 하단 토스트가 함께 표시되도록 개선했습니다.

## 7단계: 빈 데이터 화면에 행동 버튼 추가

- `StateBlock` 컴포넌트에 슬롯을 추가해 빈 상태 화면 안에 버튼을 넣을 수 있게 했습니다.
- 빈 데이터 상태에서 사용자가 바로 다음 행동을 할 수 있도록 버튼을 추가했습니다.
- 적용 예시:
  - 식단 없음: `음식 추가하기`, `음식 데이터 보기`
  - 운동 기록 없음: `운동 기록하기`, `운동 찾기`
  - 신체 기록 없음: `신체 정보 입력하기`, `진행 현황 보기`
  - 게시글 없음: `검색 초기화`, `게시글 작성하기`
  - 댓글 없음: `댓글 작성하기`
  - 음식 검색 결과 없음: `검색 초기화`, `식단 기록하기`
  - 운동 검색 결과 없음: `검색 초기화`, `운동 기록하기`
  - 진행 현황 데이터 없음: `식단 기록하기`, `운동 기록하기`, `신체 기록하기`

## 8단계: 운동/음식 카드 정보 밀도 조정

- 음식 검색 카드의 정보를 더 비교하기 쉽게 정리했습니다.
  - `100g 기준` 표시
  - 칼로리 강조
  - 탄수화물, 단백질, 지방 g 단위 표시
  - 탄단지 비율 막대 표시
  - `식단에 기록하기` 버튼 추가
- 운동 목록 카드를 더 실사용 중심으로 개선했습니다.
  - 부위/장비 상단 압축 표시
  - 타깃 근육, 보조 근육, 동작 단계 수 표시
  - 운동 설명은 일부만 보여주고 상세 화면으로 유도
  - `상세 보기`, `기록하기` 버튼을 카드 하단에 정렬

## 9단계: 모바일 입력 폼 개선

- 모바일에서 입력 폼이 1열로 자연스럽게 접히도록 CSS를 보강했습니다.
- 입력 높이를 모바일 기준 48px 이상으로 맞춰 터치하기 쉽게 했습니다.
- 주요 숫자 입력에 `inputmode`를 적용했습니다.
  - 프로필 나이, 키, 몸무게
  - 식단 섭취량
  - 신체 기록 몸무게, 체지방률, 골격근량
  - 운동 기록 시간, 세트 수, 반복 수
- 프로필 폼과 운동 기록 폼의 깨진 문구를 정리하고 모바일 친화적으로 재구성했습니다.
- 모바일에서 저장 버튼 영역이 더 쉽게 보이고 누를 수 있도록 보강했습니다.

## 10단계: 진행 현황 그래프 고도화

- 진행 현황 화면을 새로 정리하고 그래프 표현을 강화했습니다.
- 기존 단일 칼로리 막대그래프를 `섭취 칼로리 + 운동 시간` 복합 막대그래프로 개선했습니다.
- 그래프 범례를 추가했습니다.
- 기간 요약 카드 영역을 추가했습니다.
  - 기록된 날
  - 평균 섭취 칼로리
  - 최고 섭취일
- 체중 변화 추이 그래프를 추가했습니다.
- 모바일에서 그래프가 찌그러지지 않도록 가로 스크롤과 반응형 스타일을 보강했습니다.

## 검증 내역

- 각 단계 작업 후 프론트엔드 빌드를 확인했습니다.
  - `npm.cmd run build`
- 백엔드 기본 상태도 함께 확인했습니다.
  - `python manage.py check`
- 주요 수정 화면에서 깨진 한글 문자열이 남아 있는지 검색하며 정리했습니다.

## 현재 상태 요약

- UI/UX 개선 10단계는 모두 완료되었습니다.
- 백엔드 연동 전에도 화면 흐름과 발표용 시각 완성도가 높아진 상태입니다.
- AI 기능은 아직 실제 연동 전 상태이며, 화면에서는 `AI 준비 중`으로 구분되어 있습니다.
- 추후 백엔드 API 응답 형식이 확정되면 API 모듈과 어댑터 중심으로 연결 작업을 진행하면 됩니다.
